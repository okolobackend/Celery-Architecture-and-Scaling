# Изучение библиотеки Celery. Архитектура и масштабирование

Идея посмотреть, как выглядит механизм изнутри родилась давно, ещё в рабочие будни,
когда сталкивался с проблемой того, что у клиентов не работает то одно, то другое, 
но никаких артефактов или задокументированного аномального поведения не было.
Плюс развитие контейнеризации и автомасштабирование celery порождало вопросы, а как оно внутри другого.

Тогда времени и сил не хватало, поэтому сейчас решил взяться за дело основательно.
Предварительный набег на процессы внутри докера продемонстрировал две вещи:
а) тема очень серьезная и объемная;
б) нехватка собственных знаний и методологии для работ подобного рода.

Медленно, но верно, исправляя недостатки и развивая достоинства, решил разделить на посильные этапы:
1. Принципы работы autoscale.
2. Архитектура воркера как рабочей единицы.
3. Потоки(greenlet) vs процессы(prefork).
4. Canvas Workflow, о котором прежде даже не слышал.
5. Работа в контейнере. Кубер.
6. И побочные системные явления: контекстные переключения, CoW, асинхронность, етц.

## Структура проекта
- docs
  - 01_articles -- статьи по вопросу
  - 02_runtime  -- логи, чьи метрики вошли в статью
  - 04_helps    -- всякие дополнения вокруг темы
- src
  - celery_models -- celery tasks
  - monitoring    
  - run_scripts   -- баш-скрипты, стартующие эксперименты
  - tools         -- манипуляции с созданием директорий для хранения логов 
  - utils         -- скрипты для сбора статистики
